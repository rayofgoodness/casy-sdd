# Handoff-блок — що друкує скіл наприкінці (контракт виводу)

> Reference-only. Не скіл. Кожен `write-*` скіл наприкінці друкує цей блок останнім виводом. Прибирає «не знаю що далі» для соло-юзера.

## Формат

````md
## ✅ <skill> — <feature>

**Що зробив**
- <1–3 пункти: який артефакт створив/змінив + запропонований коміт>

**Перевір**
- `specs/<feature>/<file>` — <що глянути>

**Далі**
```
/casy-sdd:<next> <feature>
```
````

## Правила

- Реальні шляхи (клікабельні/копіювані), не описи: `specs/<feature>/spec.md`, не «спека».
- Наступна команда у fenced-блоці — копіюється в один клік. Підстав реальний slug, не літерал `<feature>`.
- Друкується РІВНО раз, останнім. Forward-ланцюг: spec → plan → tasks → (team або orchestrator).
