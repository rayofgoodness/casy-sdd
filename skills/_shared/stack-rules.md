# Stack-правила Casy (канонічне джерело)

> Reference-only. Не скіл. Канонічні правила стеку. Агенти й скіли тримають короткий essential + вказівник сюди; повна версія — тут. Скіли читають цей файл за відносним шляхом; агенти дублюють essential інлайн (їхній контекст не гарантує читання plugin-файлів).

## Backend (Casy)

- **Knex обовʼязково. ORM заборонено** — без Sequelize/TypeORM/Prisma. Запити лише через Knex query builder.
- Схема БД — лише через Knex-міграції. Жодних ручних змін схеми поза міграціями.
- Помилки/валідація — за конвенціями проєкту.

## Frontend (Casy client / Pink Code)

- Наявний фреймворк, структура компонентів, патерни стану — за проєктом.
- Інтеграція з API — за контрактами з `plan.md`; ендпоінти не вигадувати.

## Спільне

- **Знайди аналог перед написанням**: Grep/Read поряд, повтори стиль і конвенції. Не вигадуй власний підхід.
- Ключ `knex_required` у `.casy-sdd.local.md` (дефолт `true`) може послабити Knex-правило для не-Casy проєктів.
